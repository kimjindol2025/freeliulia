## 🎯 Step 1: 마이크로 컴파일러

### 구성

```
마이크로컴파일러.fl
├─ Tokenizer (간단한 버전)
│  └─ 숫자, 식별자, 연산자 인식
├─ Parser (재귀 하강)
│  └─ 숫자, 변수, 함수 호출
├─ Codegen (C 코드 생성)
│  └─ 기본 C 템플릿
└─ Main
   └─ 입력 처리 & 출력
```

### 목표 기능

| 기능 | 예제 | 상태 |
|------|------|------|
| 숫자 | `10`, `42` | ✅ |
| 변수 | `x = 10` | ✅ |
| 산술 | `x + y`, `10 * 20` | ⏳ |
| 함수 | `함수 더하기() ... 끝` | ⏳ |
| if 문 | `만약 ... 끝` | ⏳ |

### 코드 구조

```freeliulia
# 마이크로 렉서
함수 토크나이즈_마이크로(source: 문자열) -> 정수
    # 토큰 카운트 반환
끝

# 마이크로 파서
함수 파싱_마이크로(tokens) -> 정수
    # AST 구조 반환 (간단화)
끝

# 마이크로 코드 생성기
함수 코드생성_마이크로(ast) -> 문자열
    # C 코드 반환
끝

# 통합
함수 컴파일_마이크로(source: 문자열) -> 문자열
    변수 tokens = 토크나이즈_마이크로(source)
    변수 ast = 파싱_마이크로(tokens)
    변수 c_code = 코드생성_마이크로(ast)
    반환 c_code
끝
```

## 📊 예상 코드량

| 컴포넌트 | Julia 버전 | FreeLiulia 버전 | 비율 |
|---------|-----------|-----------------|------|
| Tokenizer | 563줄 | 150줄 | 27% |
| Parser | 842줄 | 300줄 | 36% |
| Lowering | 435줄 | 200줄 | 46% |
| Type Infer | 520줄 | 0줄 | 0% |
| Codegen | 380줄 | 200줄 | 53% |
| **Total** | **2,740줄** | **850줄** | **31%** |

**예상 자체호스팅 컴파일러**: ~850줄 (원본의 31%)

> 더 작은 이유: 간단한 코드 생성, 타입 추론 생략, 최적화 없음

```

     ↓ [gcc 컴파일]
micro_compiler_v2 (바이너리)
```

### 3단계: 출력 일치 확인
```
if micro_compiler.c == micro_compiler_v2.c {

}
```
