# 🚀 FreeLiulia 언어 스펙

**새로운 프로그래밍 언어**: FreeLang (한글) + Julia (기술) 혼합

---

## 📖 언어 설계 철학

```
FreeLang의 장점          Julia의 장점           합체 결과
─────────────────────────────────────────────────────────
✅ 간단한 문법        ✅ 다중 디스패치      ✨ 강력하면서 단순
✅ 한글 지원          ✅ JIT 컴파일        ✨ 빠르고 접근성 높음
✅ 빠른 학습          ✅ 높은 성능         ✨ 생산성 + 성능
✅ 명확한 오류        ✅ 과학 계산 최적화  ✨ 모든 분야 지원
```

## 🎯 FreeLiulia 핵심 특징

### 1️⃣ 다중 디스패치 (Multiple Dispatch)
```
계산하기(x: 정수, y: 정수) = x + y
계산하기(x: 실수, y: 실수) = x + y
계산하기(x: 문자열, y: 문자열) = x ++ y  // 문자열 연결
```

### 2️⃣ 한글 문법 + 타입 시스템
```
함수 더하기(가: 정수, 나: 정수) -> 정수
    반환 가 + 나
끝

변수 결과: 정수 = 더하기(3, 4)
출력(결과)  // 7
```

### 3️⃣ JIT 컴파일 성능
```
반복 i = 0 부터 1000000 까지
    계산하기(i, i+1)
끝
// 네이티브 속도 with 고수준 문법
```

### 4️⃣ 배열 및 벡터화
```
배열 arr = [1, 2, 3, 4, 5]
배열 결과 = arr .* 2  // 요소별 연산
출력(결과)  // [2, 4, 6, 8, 10]
```

## 📝 FreeLiulia 문법

### 기본 타입
```
정수:       42, -10, 0
실수:       3.14, 1.5e-3, 2.0
논리값:     참, 거짓
문자열:     "안녕하세요", 'Hello'
없음:       없음
```

### 변수 선언
```
변수 x: 정수 = 10
변수 y: 실수 = 3.14
상수 PI: 실수 = 3.141592653589793
```

### 함수 정의 (다중 디스패치)
```
함수 더하기(x: 정수, y: 정수) -> 정수
    반환 x + y
끝

함수 더하기(x: 실수, y: 실수) -> 실수
    반환 x + y
끝

함수 더하기(x: 문자열, y: 문자열) -> 문자열
    반환 x ++ y  // 문자열 연결
끝

// 호출 시 타입에 맞게 자동 선택
출력(더하기(3, 4))           // 7 (정수 버전)
출력(더하기(3.14, 2.0))      // 5.14 (실수 버전)
출력(더하기("안", "녕"))      // "안녕" (문자열 버전)
```

### 제어흐름
```
만약 x > 10
    출력("큼")
아니면
    출력("작음")
끝

반복 i = 1 부터 5 까지
    출력(i)
끝

동안 x < 100
    x = x * 2
끝
```

### 배열 및 딕셔너리
```
배열 arr = [1, 2, 3, 4, 5]
실수배열 nums: 배열<실수> = [1.5, 2.5, 3.5]

딕셔너리 사람 = {
    "이름": "김철수",
    "나이": 30,
    "직업": "개발자"
}

출력(사람["이름"])  // "김철수"
```

### 메타프로그래밍
```
@인라인  // 인라인 최적화
함수 빠른계산(x: 정수) -> 정수
    반환 x * 2
끝

@벡터화  // 벡터화
함수 제곱(x: 실수) -> 실수
    반환 x * x
끝
```

## 🏗️ FreeLiulia 컴파일 파이프라인

```
FreeLiulia 소스코드
    ↓
┌─────────────────────────────┐
│ 1. 토크나이저 (Lexer)        │
│    - 한글 키워드 인식        │
│    - 정수/실수/문자열 파싱   │
└─────────────┬───────────────┘
              ↓
토큰 스트림
    ↓
┌─────────────────────────────┐
│ 2. 파서 (Parser)             │
│    - 한글 문법 파싱          │
│    - AST 생성               │
└─────────────┬───────────────┘
              ↓
AST (추상 구문 트리)
    ↓
┌─────────────────────────────┐
│ 3. 로워링 (Lowering)         │
│    - 제어흐름 정규화         │
│    - IR 생성                 │
└─────────────┬───────────────┘
              ↓
Untyped IR
    ↓
┌─────────────────────────────┐
│ 4. 타입 추론                 │
│    - 다중 디스패치 분석      │
│    - SSA IR 생성             │
└─────────────┬───────────────┘
              ↓
Typed SSA IR
    ↓
┌─────────────────────────────┐
│ 5. 최적화                    │
│    - 상수 전파              │
│    - 벡터화                  │
└─────────────┬───────────────┘
              ↓
최적화된 IR
    ↓
┌─────────────────────────────┐
│ 6. 코드 생성                 │
│    - C 코드 또는 LLVM IR    │
└─────────────┬───────────────┘
              ↓
네이티브 기계어
```

## 🎨 FreeLiulia 코드 예제

### 예제 1: 소수 찾기
```
함수 소수인가(n: 정수) -> 논리값
    만약 n < 2
        반환 거짓
    끝

    반복 i = 2 부터 n 까지
        만약 n % i == 0
            반환 거짓
        끝
    끝

    반환 참
끝

반복 num = 1 부터 20 까지
    만약 소수인가(num)
        출력(num)
    끝
끝
```

### 예제 2: 벡터 연산
```
배열 벡터1: 배열<실수> = [1.0, 2.0, 3.0]
배열 벡터2: 배열<실수> = [4.0, 5.0, 6.0]

함수 내적(v1: 배열<실수>, v2: 배열<실수>) -> 실수
    변수 합계: 실수 = 0.0
    반복 i = 1 부터 길이(v1) 까지
        합계 = 합계 + (v1[i] * v2[i])
    끝
    반환 합계
끝

출력(내적(벡터1, 벡터2))  // 32.0
```

### 예제 3: 다중 디스패치
```
함수 처리하기(x: 정수) -> 문자열
    반환 "정수: " ++ 문자(x)
끝

함수 처리하기(x: 실수) -> 문자열
    반환 "실수: " ++ 문자(x)
끝

함수 처리하기(x: 배열<정수>) -> 문자열
    반환 "배열의 합: " ++ 문자(합(x))
끝

출력(처리하기(42))           // "정수: 42"
출력(처리하기(3.14))         // "실수: 3.14"
출력(처리하기([1, 2, 3]))    // "배열의 합: 6"
```

### 예제 4: 클래스/구조체
```
구조체 점
    x: 실수
    y: 실수
끝

함수 거리(p1: 점, p2: 점) -> 실수
    변수 dx: 실수 = p1.x - p2.x
    변수 dy: 실수 = p1.y - p2.y
    반환 제곱근(dx*dx + dy*dy)
끝

변수 점1: 점 = 점{x: 0.0, y: 0.0}
변수 점2: 점 = 점{x: 3.0, y: 4.0}
출력(거리(점1, 점2))  // 5.0
```

## 🎯 구현 로드맵

```
Phase 1: 언어 정의 ✅
  ├─ 문법 정의
  ├─ 타입 시스템
  └─ 표준 라이브러리

Phase 2: 컴파일러 구현 ⏳
  ├─ Lexer (한글 토크나이저)
  ├─ Parser (한글 파서)
  ├─ Lowering (IR 생성)
  ├─ Type Inference (다중 디스패치)
  ├─ Codegen (C 또는 LLVM)
  └─ 최적화

Phase 3: 자체 호스팅 ⏳
  ├─ FreeLiulia → C 컴파일러
  ├─ C 컴파일러로 자신 컴파일
  └─ 부트스트래핑 완성

Phase 4: 확장 ⏳
  ├─ 표준 라이브러리 확충
  ├─ 성능 최적화
  └─ 생태계 구축
```
